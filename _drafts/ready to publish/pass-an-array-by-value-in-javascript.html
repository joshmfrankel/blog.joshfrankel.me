OOP JavaScript pass array by value

When using object-oriented programming with JavaScript, the this keyword refers to the current class or object. Sometimes it is necessary to have a copy of a class property that is passed by value and not be reference so that you do not alter the original property. I found this out while working with arrays that need duplicate an existing array. You can utilize the .slice(0) method to accomplish this.

Basically what .slice(0) does is attempt to run the slice method with the argument 0. This essentially tells it to slice the array at index 0 or more simply don't do anything. The side effect of doing this is that slice always returns a new array so now you will have a duplicate array of the original.

Pass by Reference vs. Pass by Value

For the longest time, these terms were always confusing to me. The phrasing seemed so foreign to them. I imagine that other people have had similiar experiences as well, so let me start by saying the difference between the two is actually quite simple. {% highlight javascript %} var deckCollection = this.deck.collection.slice(0); {% endhighlight %}
